# 동적 클래스 로딩 및 실행 시스템

## 시스템 개요
- **목표**: 주기적으로 새로운 JAR 파일을 동적으로 로드하고, 특정 클래스를 실행하는 시스템
- **주요 구성 요소**:
  - **DynamicClassTest**: 실행을 시작하는 메인 클래스
  - **DynamicClassHandler**: 클래스 및 메서드 로딩 및 실행 담당
  - **SampleClass**: 실행할 샘플 클래스

## 시스템 흐름
### 클래스 로딩 및 메서드 호출 흐름
1. **DynamicClassTest**에서 작업 예약 (ScheduledExecutorService)
2. **DynamicClassHandler**에서 JAR 파일 탐색 및 로드
3. 로드된 클래스의 메서드 실행
4. 실행 결과를 출력

### 주기적인 작업 실행
- 3초마다 새로운 JAR 파일을 확인하고 로드

## DynamicClassTest 설명
- **핵심 역할**: 주기적으로 작업을 실행하여 클래스를 로드하고 메서드를 호출
  - **scheduleTask()**: 주기적 실행을 위한 작업 예약
  - **createInputMap()**: 메서드 실행에 필요한 입력 데이터를 생성
  - **printOutput()**: 실행 결과 출력

## DynamicClassHandler 설명
- **핵심 역할**: 동적 클래스 로딩 및 메서드 호출
  - **handleDynamicClass()**: 동적으로 클래스를 로드하고 메서드 실행
  - **updateJarFilePath()**: 최신 JAR 파일을 찾고 로드
  - **loadDynamicClass()**: JAR 파일에서 클래스를 로드
  - **invokeMethod()**: 메서드를 실행하고 결과를 반환

## SampleClass 설명
- **핵심 역할**: 실행할 샘플 클래스
  - **myMethod1()**: 입력 데이터를 처리하고, 현재 시간과 날짜를 출력, 랜덤한 곱셈 결과와 별을 출력

## 주요 기술
- **Java Reflection**: 클래스와 메서드를 런타임 시 동적으로 로드하고 실행
- **ClassLoader**: JAR 파일에서 클래스를 동적으로 로드
- **ScheduledExecutorService**: 주기적인 작업 실행
- **JAR 파일 처리**: 파일 디렉토리를 순회하여 최신 JAR 파일을 동적으로 로드

## 코드 실행 흐름
1. **DynamicClassTest**에서 **DynamicClassHandler**를 통해 주기적으로 작업 예약
2. 최신 JAR 파일을 확인하고, 해당 파일에서 클래스를 로드
3. 로드된 클래스에서 메서드를 동적으로 실행하고, 실행 결과 출력

## 주요 출력 예시
- **SampleClass 실행 결과**:
  - **날짜와 시간**: 현재 날짜와 시간 출력
  - **입력 값 처리**: `argument1`과 `argument2` 값 출력
  - **랜덤한 결과**: 곱셈 결과와 함께 랜덤한 별 출력

## 시스템의 유용성
- **동적 확장성**: 새로운 JAR 파일을 동적으로 로드하여 확장 가능
- **유지보수 용이성**: 메서드를 추가하거나 수정할 때 프로그램을 재컴파일하지 않고 동적으로 반영
- **유연성**: 다양한 클래스를 동적으로 실행하고, 입력 데이터를 유연하게 처리

## 향후 개선 사항
- **에러 처리 개선**: 예외 처리 및 로깅을 통한 안정성 향상
- **성능 최적화**: 클래스 로드 및 메서드 실행 성능 개선
- **확장성 향상**: 다양한 JAR 파일과 클래스를 효율적으로 관리

### 주된 차이점:
1. **클래스 로딩 방식**:
   - `DynamicClassHandler`는 JAR 파일을 직접 찾아서 로드하는 방식으로, 동적으로 클래스를 로드하고 메서드를 호출합니다.
   - Java HotSwap과 Spring Boot DevTools는 이미 실행 중인 애플리케이션에서 클래스를 변경하고 이를 자동으로 반영합니다.

2. **메서드 호출 및 리소스 반영**:
   - `DynamicClassHandler`는 메서드 호출을 명시적으로 처리하고 동적으로 로드된 클래스를 사용합니다.
   - Java HotSwap은 애플리케이션 실행 중에 수정된 클래스를 즉시 반영합니다.
   - Spring Boot DevTools는 개발 시 코드 변경을 감지하고 서버를 재시작하지 않고 자동으로 반영합니다.

3. **사용 용도 및 설정**:
   - `DynamicClassHandler`는 특정 JAR 파일의 로드 및 메서드 호출을 제어하는 데 사용되며, 특정 디렉토리에서 JAR 파일을 찾아서 동작합니다.
   - Java HotSwap은 실행 중인 애플리케이션의 클래스를 수정하고 핫스왑을 통해 반영하는 데 사용됩니다.
   - Spring Boot DevTools는 Spring Boot 애플리케이션에서 개발 중 코드 변경을 자동으로 감지하고 반영하는 용도로 사용됩니다.
